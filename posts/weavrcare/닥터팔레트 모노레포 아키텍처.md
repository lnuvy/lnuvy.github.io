#### 닥터팔레트 모노레포 및 마이크로 프론트엔드 아키텍처 구축

- **React 19, Rspack, Module Federation 기반 마이크로 프론트엔드**:

  - Rspack + Module Federation으로 70개 이상의 기능 패키지(p_planner, p_d_day, p_text_editor, p_statistic, p_pixink_web, p_plan_history, p_plan_reservation, p_partners_plan 등)를 독립적으로 빌드·배포할 수 있는 마이크로 프론트엔드 아키텍처를 운영합니다.
  - 모듈 간 순환참조 문제를 해결하고, `exposes` 컴포넌트를 각 모듈로 분리하여 MF 경계(MFBoundary)를 명확히 정의하는 컨벤션을 정립했습니다. (예: PlanModalCreate/Edit, PlannerContainer 등에서 발생하는 Module Federation 원격 모듈 로딩 에러(remoteError) 방지 구조 설계)
  - Turborepo의 캐싱 전략과 태스크 파이프라인을 최적화하여 CI 빌드 시간을 단축했습니다.
  - 새로운 도메인(D-Day, 플랜이력, 예약관리, 웹 캔버스) 요구사항에 맞춰 신규 MF 모듈을 설계·추가하고, 기존 모듈과의 의존성을 최소화하는 아키텍처를 유지했습니다.

- **Apollo Client & GraphQL Subscription**:

  - Apollo Cache의 `keyArgs`, `merge` 정책을 도메인별로 설계하여, 무한스크롤(fetchMore)과 커넥션 기반 페이지네이션의 캐시 충돌 문제를 해결했습니다. (통계-차트, 통계-진료, 이미지 커넥션 등)
  - `useLazyQuery` 사용 시 GC에 의해 캐시가 사라지는 이슈를 프로젝트 전역에서 `retain()` 패턴으로 일괄 수정하여 안정성을 확보했습니다. (알림, 메디톡, 차트뷰, 내원이력 등 다수 모듈)
  - WebSocket 기반 GraphQL Subscription을 활용해 플래너, 대기리스트, D-Day 등 실시간 데이터 동기화가 필요한 도메인을 구현했습니다. 서버 subs 이벤트에 따른 불필요한 refetch를 방지하는 최적화 패턴을 적용했습니다.
  - `useSuspenseQuery` 도입으로 탭 전환 시 깜빡임을 방지하고, `cache-and-network` / `network-only` 등 상황별 최적의 fetch 정책을 선택하여 적용했습니다.

- **모노레포 공용 라이브러리 설계**:

  - 각 모듈에서 중복되던 훅과 유틸리티를 `@pltt/utils`로 추출하고, 공용 컴포넌트를 `palette-components`로 분리하여 코드 재사용성을 높였습니다. (플랜용 타입가드 함수, 차트 필터 훅, 날짜 파싱 유틸 등)
  - 디자인 토큰(컬러, 타이포그래피)을 `@pltt/design-tokens` 패키지로 이관하여 디자인 시스템과 비즈니스 로직 패키지 간의 의존성을 단방향으로 정리했습니다. (TagOrigin 컬러 상수, ColorPicker 등)
  - 프론트엔드-백엔드 동시 작업 시 스키마 싱크를 맞추기 위한 스크립트와 가이드 문서를 작성하여 교차 개발 환경의 DX를 개선했습니다.

- **WebAssembly(WASM) 캔버스 통합**:

  - Flutter 기반 WASM 캔버스 엔진을 웹에 통합하여, 차트 이미지에 드로잉·텍스트·도형 편집 기능을 구현했습니다. (p_pixink_web 모듈)
  - Service Worker 캐싱, 이미지 캐시 버스트, 클립보드 복사/붙여넣기 등 크로스 플랫폼 이슈를 트러블슈팅하며 안정화했습니다.
  - 캔버스 툴바 상태 동기화, 텍스트 엘리먼트 편집 모드, 숏컷 충돌 방지, 패닝/줌 UX 등 복잡한 상호작용 로직을 설계하고 20회 이상의 WASM 업데이트 사이클을 거쳐 프로덕션에 안정적으로 배포했습니다.
  - 캔버스 모드에서의 이미지 회전/반전, 빈 페이지 추가, 전후사진 비교 탭 등 도메인 특화 기능을 구현하며 기존 차트 이미지 모달과 매끄럽게 통합했습니다.

- **플래너(Planner) 코어 도메인 개발**:

  - 병원 예약·접수·대기 관리의 핵심인 플래너 모듈을 주도적으로 개발했습니다. Google 캘린더 스타일의 포지션 계산 알고리즘, 드래그앤드랍(DnD) 시간 이동, 스케줄 슬롯 관리 등 복잡한 캘린더 UI를 구현했습니다.
  - 네이버, 여신티켓, 구글 비즈니스, 메디패스, 카카오 등 다수의 외부 파트너사 예약 연동을 구현하고, 각 파트너별 플랜 모델(MedipassPlanModel, PartnerPlanModel)의 상태 흐름(요청→확정→거절→취소)을 설계했습니다.
  - D-Day 관리 모듈(p_d_day), 플랜이력 모듈(p_plan_history), 예약탭 테이블뷰(p_plan_reservation) 등 플래너 생태계를 확장하며 각 모듈 간 Subscription 기반 실시간 동기화를 구현했습니다.
